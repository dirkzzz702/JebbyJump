# Jebby Jump｜Art Bible v0.2

## 1. Purpose

This Art Bible defines the visual direction for **Jebby Jump** so that all future posters, sprites, UI, platforms, items, obstacles, backgrounds, and marketing assets stay consistent.

The goal is to build a recognizable, warm, polished indie-game identity around **Jebby the Color Knight**.

## 4. Main Character: Jebby the Color Knight

### Core Concept

Jebby is **not a dog**.

Jebby is a **Cavalier King Charles Spaniel-inspired humanoid fantasy color knight**.

He carries the warmth, gentleness, loyalty, and elegance of a Cavalier, but he is an original fantasy mascot character designed for a memory-platformer game.

### One-line Character Description

**Jebby is a brave little color knight with warm Cavalier-inspired features, soft flowing ear-like hair, a blue cape, little boots, and a rainbow gem badge.**

### Character Essence

Jebby should feel like:

```text
A tiny knight who is brave enough to climb the rainbow tower,
but gentle enough to feel like a child’s best friend.
```

---

## 5. Jebby Design Rules

### Must Keep

- Humanoid / fantasy mascot body
- Not four-legged
- Not a realistic dog
- Cavalier-inspired brown-and-white palette
- Large warm eyes
- Soft long ear-like hair or ear feathers
- Small blue cape
- Little boots
- Rainbow gem / color badge
- Friendly brave expression
- Childlike proportions
- Soft rounded silhouette

### Do Not Do

- Do not make Jebby a literal dog.
- Do not make Jebby crawl or stand on four legs.
- Do not give Jebby realistic dog snout anatomy.
- Do not make Jebby look like an adult warrior.
- Do not use heavy medieval armor.
- Do not make the design too complex for animation.
- Do not change his core brown-white-blue-rainbow identity.

---

## 6. Character Proportions

Recommended Jebby proportions:

```text
Overall style: chibi / small fantasy mascot
Head-to-body ratio: around 2.5 heads tall
Head: large and rounded
Eyes: large, warm, expressive
Body: small and compact
Arms: short but expressive
Legs: short, suitable for jumping animation
Feet: slightly oversized boots for readability
Cape: small, readable silhouette
Ear-like hair: long and soft, moving with jumps
```

### Silhouette Priority

Jebby should be recognizable even in a small sprite.

Important silhouette features:

1. Round head
2. Long side ear-feathers / hair
3. Small cape
4. Little boots
5. Upright heroic pose

## 8. Facial Expression Rules

### Default Expression

- Warm
- Curious
- Brave
- Slight smile
- Big eye highlights

### Gameplay Expressions

| State | Expression |
|---|---|
| Idle | Calm, friendly, ready |
| Run | Focused but happy |
| Jump | Excited, determined |
| Fall | Slightly surprised |
| Correct landing | Proud / delighted |
| Wrong landing | Dizzy / embarrassed, not tragic |
| Hurt | Soft comic discomfort, not scary |
| Victory | Big smile, arms up |
| Shop / item | Curious, sparkling eyes |

### Tone Rule

Even when Jebby fails, the expression should feel safe and playful.

No horror, pain, or intense fear.

## 10. Wardrobe / Outfit System Roadmap

### Product Decision

The outfit system should be included in the long-term plan, but **not in MVP**.

```text
MVP: Default Jebby only
V2: Basic outfit preview or 1–2 unlockable cosmetic outfits
V3: Full Jebby’s Wardrobe system with outfit fragments
```

### System Name

Recommended name:

```text
Jebby’s Wardrobe
```

Alternative names:

```text
Color Knight Wardrobe
Rainbow Wardrobe
Knight Outfits
```

### Default Character

The default character is:

```text
Classic Cavalier / Jebby the Color Knight
```

This is Jebby’s core identity, not just a skin.

### Future Cosmetic System

Players may collect outfit fragments. When all fragments of an outfit are collected, the outfit is unlocked.

Example:

```text
Forest Cavalier Outfit
- Forest Cape Fragment
- Leaf Brooch Fragment
- Green Boots Fragment
- Forest Tunic Fragment
- Nature Feather Fragment
```

### Initial Outfit Candidates

Recommended first outfit candidates:

1. Forest Cavalier
2. Sunshine Knight
3. Aqua Knight
4. Silver Dreamer

Later candidates:

- Crimson Hero
- Rookie Page
- Pastel Prince
- Seasonal outfits

### Gameplay Rule

Early outfits should be **cosmetic first**.

Do not introduce gameplay advantages until the core loop and balance are stable.

If gameplay bonuses are added later, they should be light and non-pay-to-win.

Examples of possible later soft bonuses:

```text
Forest Cavalier: small one-time fall recovery visual effect
Aqua Knight: slightly reduced ice slide
Sunshine Knight: tiny memory-time bonus
Silver Dreamer: cosmetic sparkle trail only
```

### Outfit Design Rule

Outfits may change:

- Cape color
- Clothing theme
- Boots
- Trim
- Small accessories
- VFX color

Outfits must preserve:

- Jebby’s face identity
- Chibi proportions
- Brown/ivory or recognizable Cavalier-inspired hair structure
- Warm eyes
- Long ear-feather silhouette
- Rainbow gem motif
- Friendly brave personality

### Claude / AI Rule

Do not implement wardrobe, skins, outfit fragments, cosmetic inventory, or outfit economy until explicitly approved.

For now, wardrobe is a **future roadmap system only**.

- Dreamlike vertical space

### World Themes

2. Cloud Valley

## 13. Platform Style

### Core Platform Design

Platforms should be:

- Rounded rectangles or soft floating stones
- Clearly color-coded
- Easy to read at speed
- Not overly textured
- Slight magical glow or outline

### Platform Color Rule

The platform color must be immediately readable.

Recommended platform colors:

```text
Red
Blue
Green
Yellow
Purple
Orange
```

### Visual Treatment

Each platform may include:

- Main color fill
- Slight highlight on top edge
- Soft shadow underneath
- Subtle glow
- Optional icon/symbol for accessibility

### Accessibility Option Later

Each color can also map to a symbol:

```text
Red = star
Blue = moon
Green = leaf
Yellow = sun
Purple = diamond
Orange = flame
```

This helps color-blind players and young children.

- Rounded
- Soft
- Friendly
- Clear

- Magical color accents

### UI Shape Language

Use:

- Rounded panels
- Pill buttons
- Soft shadows
- Light borders
- Small sparkle accents

- Dense text

## 20. Unity Import Guidelines

### Sprite Import

Recommended default:

```text
Texture Type: Sprite (2D and UI)
Sprite Mode: Single or Multiple depending on sheet
Pixels Per Unit: choose consistently after prototype scale is stable
Filter Mode: Bilinear for painterly art, Point only for pixel art
Compression: Normal Quality for production, None during testing if artifacts appear
Generate Mip Maps: usually off for 2D sprites
```

### Transparent Assets

Character sprites and item icons should use transparent PNG.

### Sprite Atlas Later

Use Sprite Atlas later for:

- Character sprites
- Platform sprites
- UI icons
- Item icons

Do not optimize too early.

## 22. Consistency Checklist

Before accepting any generated asset, check:

```text
Does Jebby still look like Jebby the Color Knight?
Is he humanoid/fantasy, not a literal dog?
Does he keep brown + ivory + blue cape + rainbow badge?
Are the eyes warm and friendly?
Are the ear-like hair/ear feathers present?
Is the silhouette readable at small size?
Does the style match soft polished 2D indie game art?
Can this be animated or used in Unity?
Is the background too noisy for gameplay?
Are colors consistent with the palette?
Does it avoid scary/adult/gritty design?
```

If an asset fails more than two checks, regenerate or revise it.

---

## 23. Production Recommendation

For the most consistent final result:

```text
AI generates concept and draft assets.
Human review selects the best version.
Final pass cleans shapes, colors, transparency, and animation readiness.
Unity uses consistent import settings and naming rules.
```

AI is excellent for exploration and drafts, but final game sprites may still need manual cleanup.
